## 2.0.0 - 2026-03-16
* The `searchsimilar` client module has been removed from the SDK. All methods including `searchsimilar.get()` and `searchsimilar.post()` are no longer available. Applications using these methods will need to migrate to alternative search functionality or use the core search endpoints directly.
* The RawSearchsimilarClient and AsyncRawSearchsimilarClient classes have been removed from the SDK. If your code directly imports or uses these raw client classes, switch to the standard SearchsimilarClient which provides the same functionality with a cleaner interface.
* The Search Similar functionality has been removed from the SDK, including all related types and client methods. This feature was deprecated and no longer supported by the API. Code using SearchSimilar endpoints or related types will need to be updated to use alternative search methods.
* The aggregation client methods have been renamed for clarity: `get()` is now `count_get()` and `post()` is now `count_post()`. These methods continue to function identically but with more descriptive names. Update your code to use the new method names, as the old names have been removed.
* The aggregation client methods have been renamed for clarity. The `get()` method is now `count_get()` and the `post()` method is now `count_post()`. Update your code to use the new method names - all parameters and return types remain the same.
* The SDK now supports lazy loading to improve startup performance and reduce memory usage. Clients are initialized only when accessed, reducing initial import time. Added optional logging configuration to base client constructors for better debugging capabilities.
* The HTTP client now features enhanced retry logic with intelligent rate-limit handling, comprehensive request/response logging capabilities, and improved URL building for tenant-based endpoints. A new status_code property is available on HTTP response objects.
* The SDK now includes enhanced Server-Sent Events parsing capabilities and improved Pydantic model field handling. These improvements provide better support for streaming APIs and more flexible input handling for model fields.
* The latest headlines client has been renamed and restructured. The module path has changed from `latestheadlines` to `latest_headlines`, and the client class is now called `LatestHeadlinesClient` instead of `LatestheadlinesClient`. Method names have changed from `get()` and `post()` to `latest_headlines_get()` and `latest_headlines_post()`. Update your imports from `client.latestheadlines` to `client.latest_headlines` and update method calls accordingly.
* The latest headlines client has been restructured with breaking changes to class and method names. The `RawLatestheadlinesClient` class is now `RawLatestHeadlinesClient`, and methods `get()` and `post()` are now `latest_headlines_get()` and `latest_headlines_post()` respectively. Additionally, many parameters now use strongly-typed classes instead of primitive types. Update your imports and method calls accordingly.
* The search_link module has been renamed to search_by_link. Update imports from `client.search_link` to `client.search_by_link`. Method names have also changed: `search_url_get()` is now `search_by_link_get()` and `search_url_post()` is now `search_by_link_post()`. Parameter types are now more strictly typed (Page, PageSize, RobotsCompliant) instead of primitive int/bool types.
* Added support for the new `all_links_text` parameter in Authors API methods, allowing searches based on link anchor text content. Parameter types have been updated to use proper type aliases for improved type safety.
* The breaking news API methods now use strongly-typed parameter classes instead of primitive types for improved type safety. The `robots_compliant` parameter has been removed from breaking news methods.
* The search API now includes a new `all_links_text` parameter for filtering articles by link anchor text. All search parameters have been enhanced with proper type definitions for improved IDE support and validation.
* The Search API now supports searching by link anchor text content with the new `all_links_text` parameter. When used, the response includes detailed link information in the `all_links_data` field.
* The SDK now includes enhanced type definitions for better API compatibility, with new translation fields available on breaking news articles when the `include_translation_fields` parameter is enabled. Server-Sent Events (SSE) support has been added for real-time data streaming capabilities.
