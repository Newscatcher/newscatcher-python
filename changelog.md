## 3.0.0 - 2026-03-16
* The aggregation API client has been renamed and reorganized for better clarity. The `client.aggregation` property is now `client.aggregation_count`, and methods have been simplified from `count_get`/`count_post` to `get`/`post`. All response types have been renamed to match. Update your code: replace `client.aggregation.count_get()` with `client.aggregation_count.get()` and `client.aggregation.count_post()` with `client.aggregation_count.post()`.
* Method names have been simplified across all client modules. Methods like `latest_headlines_get()`, `search_by_link_post()`, and `authors_get()` are now renamed to `get()` and `post()`. Update your code to use the new method names: replace `client.latest_headlines.latest_headlines_get()` with `client.latest_headlines.get()`, and similarly for all other affected methods. Response type names have also changed from service-prefixed to verb-prefixed patterns.
* Type alias names have been renamed for consistency: `SearchGetResponse` is now `GetSearchResponse`, `SearchPostResponse` is now `PostSearchResponse`, and `LatestHeadlinesPostResponse` is now `PostLatestHeadlinesResponse`. Update any direct imports or type annotations to use the new names.
